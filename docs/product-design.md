# Gabutin — High-Level Design Doc

**Date:** 2026-05-27
**Status:** Approved
**Scope:** v1.0 (Hackathon — 48h, deploy to Google Cloud Run)

---

## 1. Problem Statement

Gen-Z Indonesia scroll TikTok dan IG berjam-jam tapi nggak dapet apa-apa. Gabutin pakai mekanik scroll yang sama tapi isinya tantangan pengetahuan — fakta singkat dari Wikipedia diikuti soal MCQ. Dopamine loop-nya sama, tapi otak dapet sesuatu yang nyata. Target: Gen-Z Indonesia usia 16–25, mobile-first tapi bisa dibuka lewat desktop, familiar dengan budaya internet Indonesia.

---

## 2. Goals

- **v1.0 (Hackathon)**
 - Tutorial → onboarding → adaptive knowledge feed
 - Guest play only (display name + 9-digit ID, no account required)
 - XP system + level progression (polynomial curve)
 - Achievement system with badge showcase (sense of progression, replaces leaderboard)
 - Deploy as a single container on Google Cloud Run

- **v1.x (Post-hackathon)**
 - Weekly leaderboard + Hall of Fame
 - OAuth accounts (Google / X / Facebook)
 - User-submitted content with moderation queue
 - Native iOS/Android app

## 3. Non-Goals (v1.0)

- OAuth / social login (Google, X, Facebook)
- Leaderboard or Hall of Fame
- Content moderation UI / admin panel
- User-submitted questions
- Push notifications

## 6. Architecture

Single Next.js monolith — no separate Express backend. Server Actions handle all internal mutations. One container deployed to Cloud Run.

```
Browser (PWA)
  Next.js 16 App Router
     React Server Components (feed, achievements, profile reads)
     Server Actions (answer submission, score updates, user creation)
     Client Components (card interactions, swipe gestures, timers)

External Services
  MongoDB Atlas    — users, cards, scores, achievements
  Wikipedia REST API — article content for card facts (id.wikipedia.org)
  Gemini API     — MCQ question generation from Wikipedia excerpts
```

### Card Generation Pipeline (Lazy)

No cron job. The pipeline triggers silently whenever the card pool for a theme falls below 10 cards.

```
Feed requested by user
 → Check card pool per theme in DB
 → If pool < 10 for any of user's themes:
   → Fetch random Wikipedia article from id.wikipedia.org (background)
   → Extract intro paragraph (≤ 200 words)
   → Call Gemini API: generate 1 MCQ in Bahasa Indonesia, casual register, 4 options + correct index
   → Save card to MongoDB (status: 'approved')
 → Serve next unseen card to user from pool
```

Generation runs in the background — the user is served from the existing pool while new cards are generated. First-time users on an empty pool wait ~3s for the first card only.

## 8. User Identity

No accounts in v1.0. All users are guests.

### Flow

```
First visit (no uniqueUserId in localStorage)
 → Tutorial: 3 hand-curated cards
 → End of tutorial → "Simpan progress lo?"
   Option A: Enter display name → pick 3 themes → DB record created
   Option B: Skip → localStorage flag only (progress may be lost)

Returning visit (uniqueUserId in localStorage)
 → Load user from DB → go straight to /feed
```

### uniqueUserId Generation

```ts
const uniqueUserId = Math.floor(100_000_000 + Math.random() * 900_000_000).toString()
// always 9 digits, stored in localStorage
```

### Guest Re-engagement (for skipped users)

Condition: `guestCardCount >= 15` AND `(now - lastReminderShown) >= 24h`

Injects one swipeable reminder card into the feed (not a popup):

> "Lo udah jawab 15+ soal. Jangan sampe ilang — simpan progress lo dalam 10 detik."
> `[Simpan Progress]` `[Ntar deh →]`

Resets reminder clock on display. Repeats every 24h until user registers.

## 9. XP & Leveling System

### Level Threshold Formula

```ts
function xpRequiredForLevel(level: number): number {
 return Math.floor(10 * Math.pow(level, 1.5))
}
```

| Level | XP to next level | Total XP from scratch |
|---|---|---|
| 1 | 10 | 10 |
| 2 | 28 | 38 |
| 3 | 52 | 90 |
| 5 | 111 | 298 |
| 10 | 316 | 1,296 |
| 20 | 894 | 5,025 |
| 50 | 3,536 | 39,217 |

### XP Per Answer Formula

```ts
const BASE_XP = 2

function getStreakBonus(streak: number): number {
 if (streak >= 10) return 5
 if (streak >= 5) return 3
 if (streak >= 3) return 1
 return 0
}

function calculateXP(result: 'correct' | 'wrong' | 'skip', streak: number): number {
 if (result !== 'correct') return 0
 return BASE_XP + getStreakBonus(streak)
}
```

### XP per Correct Answer

| Current Streak | XP Gained | Breakdown |
|---|---|---|
| 1–2 | 2 | base(2) + bonus(0) |
| 3–4 | 3 | base(2) + bonus(1) |
| 5–9 | 5 | base(2) + bonus(3) |
| 10+ | 7 | base(2) + bonus(5) |
| Wrong / Skip | 0 | streak resets to 0 |

### Level Titles (Bahasa campur, Gen-Z flavor)

| Range | Title | Vibe |
|---|---|---|
| 1–5 | Newbie Gabut | Baru mulai, masih hangat |
| 6–15 | Penasaran | Mulai ketagihan belajar |
| 16–30 | Anak Pintar | Udah serius nih |
| 31–50 | Sultan Ilmu | Respect level tinggi |
| 51+ | Dewa Gabut | Gabut tapi pinter banget |

### Level Calculation

Level is recomputed from total XP on every answer. Stored in `users.level` for fast reads but always derivable from `users.xp`.

## 11. Feed Algorithm

The feed shows cards weighted toward the user's **weakest themes**.

```
1. Fetch user's theme_scores for their 3 chosen themes
  e.g. { Matematika: 8, Sejarah: 2, Pop Culture: 5 }

2. Invert scores (weakness = higher weight)
  Raw weights: Matematika = 1/(8+1), Sejarah = 1/(2+1), Pop Culture = 1/(5+1)

3. Normalize to sum = 1.0

4. Weighted random theme selection → e.g. Sejarah chosen

5. Fetch one unseen card from that theme's approved pool
  (exclude IDs in theme_scores.seenCardIds)

6. Add cardId to seenCardIds

7. If pool < 10 for selected theme → trigger background generation
```

**Score floor:** Theme points never go below 0.

## 12. Scoring Rules

| Action | Theme Points | XP | Streak |
|---|---|---|---|
| Correct (streak 1–2) | +2 | +2 | +1 |
| Correct (streak 3–4) | +2 | +3 | +1 |
| Correct (streak 5–9) | +2 | +5 | +1 |
| Correct (streak 10+) | +2 | +7 | +1 |
| Wrong | -2 | 0 | reset to 0 |
| Skip / timer expires | -1 | 0 | reset to 0 |

Theme points drive the adaptive feed algorithm. XP drives level progression. Streak drives XP multiplier. All three are independent systems.

## 13. Achievement System

Achievements are the primary progression signal in v1.0. Shown on `/achievements` as a badge grid. Unlocking one triggers an in-feed toast notification. Users can **showcase up to 3 badges** on their profile — pinned prominently as a personal identity statement.

### Badge Showcase Mechanic

- Every earned achievement has a **badge** (icon + title + rarity color)
- On `/achievements`, earned badges show a "Pin ke Profil" button
- Max 3 badges pinned at once — pinning a 4th replaces the oldest
- Profile page shows the 3 showcase slots prominently above stats
- Locked achievements shown as silhouettes with progress hints (e.g. "47/100 soal")

## 14. Analytics & Validation (PostHog)

PostHog free cloud tier — 1M events/month, built-in surveys, no self-hosting needed.

### Setup

```bash
pnpm add posthog-js posthog-node
```

```
NEXT_PUBLIC_POSTHOG_KEY   # PostHog project API key (safe to expose)
NEXT_PUBLIC_POSTHOG_HOST  # https://app.posthog.com (or eu.posthog.com)
```

Initialize once in `app/providers.tsx` (Client Component wrapper). Use `posthog-node` in Server Actions for server-side events.

### Key Events to Track

| Event | Trigger | Properties |
|---|---|---|
| `tutorial_started` | /welcome page load | — |
| `tutorial_completed` | 3rd tutorial card answered | — |
| `onboarding_completed` | User saved display name + themes | `{ themes: string[] }` |
| `onboarding_skipped` | User clicked skip | — |
| `card_viewed` | Card STATE 1 shown | `{ theme, card_id }` |
| `card_answered` | Answer submitted | `{ theme, result, xp_gained, streak, level }` |
| `card_skipped` | Skip pressed or timer expired | `{ theme, streak_lost }` |
| `level_up` | Level threshold crossed | `{ new_level, title }` |
| `achievement_unlocked` | Achievement earned | `{ key, title, rarity }` |
| `badge_showcased` | Badge pinned to profile | `{ key, position }` |
| `re_engagement_shown` | Reminder card injected | `{ cards_answered }` |
| `re_engagement_converted` | User clicked "Simpan Progress" on reminder card | — |

### In-App Surveys (PostHog native)

Configured in PostHog dashboard — no code changes needed beyond SDK initialization.

| Survey | Trigger | Goal |
|---|---|---|
| **Validation survey** | After 10th correct answer | "Setelah main Gabutin, lo ngerasa lebih sering pengen belajar hal baru?" (Y/N + optional text) |
| **NPS survey** | After 50th answer | "Seberapa besar lo bakal rekomendasiin Gabutin ke temen lo?" (0–10 scale) |
| **Theme feedback** | After 20 answers in one theme | "Soal di tema ini udah sesuai sama yang lo harapin?" (Y/N) |
| **Churn survey** | On app close after 3+ day gap | "Lo kemana aja? Apa yang bikin lo balik?" (multiple choice) |

### Validation Metrics to Watch

- **Tutorial → onboarding conversion rate** — are users registering after the tutorial?
- **Re-engagement card conversion rate** — does the reminder card actually work?
- **Session length + cards per session** — are users staying?
- **Theme distribution** — which themes are most played / most skipped?
- **Streak distribution** — are users hitting streak bonuses or bouncing early?

## 15. UI Structure

### Routes

```
/           → Redirect to /feed (or /welcome if first visit)
/welcome       → Tutorial (3 cards) + registration prompt
/feed         → Main scroll feed
/feed/[cardId]    → Deep-link to specific card (shareable)
/achievements     → Full badge grid (earned + locked silhouettes), pin to showcase
/profile       → Badge showcase (3 slots) · Display name · User ID · Level + XP bar · Theme breakdown
```

### Card Lifecycle (4 states)

```
STATE 1: Fact
 → Wikipedia excerpt displayed
 → Source URL link
 → "Siap?" CTA or auto-advance after 5s

STATE 2: Question
 → Question text
 → 4 MCQ option buttons
 → 10-second countdown timer
 → Timer expires = treated as skip (-1 theme point, 0 XP, streak reset)

STATE 3: Result
 → Correct/wrong indicator
 → Theme points delta (+2 / -2 / -1)
 → XP gained if correct (+2 to +7 XP depending on streak, level-up animation if threshold crossed)
 → Achievement toast if newly unlocked
 → Brief explanation (1–2 sentences in Bahasa Indonesia)

STATE 4: Next
 → Swipe up or tap "Next" to load next card
```

### Navigation

- Bottom nav (mobile persistent): Feed · Achievements · Profile
- No top nav on feed — full-screen immersive scroll
